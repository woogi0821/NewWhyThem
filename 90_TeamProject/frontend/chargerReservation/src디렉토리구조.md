# 📁 프론트엔드 디렉토리 구조 가이드 (React + TypeScript + Vite)

본 프로젝트의 프론트엔드 폴더 구조 및 각 디렉토리의 역할입니다. 
작업 시 아래의 기준에 맞춰 파일을 분리하여 생성해 주시기 바랍니다.

## 📂 폴더별 역할 요약

* **`assets/`**
  * 이미지, 폰트, 아이콘 등 화면에 보여줄 정적인 디자인 파일들을 보관하는 창고입니다.
* **`common/`**
  * 프로젝트 전체에서 공통으로 재사용할 유틸리티 함수, 상수, 전역 상태 등을 모아두는 곳입니다.
* **`components/`**
  * 버튼, 입력창, 모달 팝업처럼 여러 화면에서 레고 블록처럼 조립해서 쓸 수 있는 재사용 UI 조각들입니다.
* **`hooks/`** * 충전소 검색, 필터링 등 복잡한 상태 관리와 비즈니스 로직을 UI에서 분리해 낸 커스텀 훅 보관소입니다.쉽게설명하면 js에서 기능 구현 함수만
따로 관리하는 파일이 들어갑니다.
* **`layout/`**
  * 헤더(GNB), 푸터, 사이드바 등 웹사이트의 전체적인 뼈대와 레이아웃 틀을 담당합니다.
* **`pages/`**
  * 사용자가 URL을 타고 들어왔을 때 보게 되는 각각의 전체 화면(홈 화면, 예약 화면, 키오스크 화면 등)입니다.
* **`routers/`**
  * 사용자가 입력한 URL 주소에 맞춰 어떤 `pages/`를 보여줄지 길을 안내해 주는 네비게이션입니다.
* **`services/`**
  * 백엔드(Spring Boot) API로 데이터를 요청하고 받아오는 통신 전용 함수(Axios)들의 모음입니다.
* **`types/`**
  * 프론트엔드와 백엔드가 주고받을 데이터 모양(DTO)을 엄격하게 정의해 둔 타입스크립트 명세서입니다.
* **`validation/`**
  * 이메일 형식, 핀번호 4자리 등 사용자 입력값이 올바른지 프론트엔드 단에서 1차로 검사하는 규칙 모음입니다.
